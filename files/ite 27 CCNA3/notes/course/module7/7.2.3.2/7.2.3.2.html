Missing Network Statement

In Figure 1, R1’s GigabitEthernet 0/1 interface has just been configured with the 10.10.10.1/24 address and is active. R1 and R3 have an established EIGRP neighbor adjacency and are exchanging routing information. However, as shown in Figure 2, a ping test from the R3 router to a R1’s G0/1 interface of 10.10.10.1 is unsuccessful.

In Figure 3, the output of the show ip protocols command on the R1 router shows that the network 10.10.10.0/24 is not advertised to EIGRP neighbors. Therefore, network 10.10.10.0 must be advertised.

In Figure 4, R1’s EIGRP process is configured to include the advertisement of the classful 10.0.0.0 network.

Figure 5 confirms the addition of the 10.10.10.0/24 route in R3’s routing table. Reachability is verified and confirmed by pinging R1’s GigabitEthernet 0/1 interface.

EIGRP for IPv6

Similar commands and troubleshooting criteria also apply to EIGRP for IPv6.

The following are the equivalent commands used with EIGRP for IPv6:

  • Router# show ipv6 protocols

  • Router# show ipv6 route

To add a missing network statement under IPv6, use the ipv6 eigrp autonomous-system command in interface configuration mode:

Router(config-if)# ipv6 eigrp autonomous-system

Note: Another form of missing route may result from the router filtering inbound or outbound routing updates. ACLs provide filtering for different protocols, and these ACLs may affect the exchange of routing protocol messages. This can cause routes to be absent from the routing table. Use the show ip protocols command to see if there are any ACLs that are applied to EIGRP. Applying filters to routing updates is beyond the scope of this curriculum.
