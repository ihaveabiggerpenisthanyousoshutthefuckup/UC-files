Summary Route

The following scenario explains how automatic summarization could also cause a routing loop to occur.

In the figure:

1. R1 has a default route, 0.0.0.0/0 via the ISP router.

2. R1 sends a routing update to R2 containing the default route.

3. R2 installs the default route from R1 in its IPv4 routing table.

4. R2â€™s routing table contains the 172.16.1.0/24, 172.16.2.0/24, and 172.16.3.0/24 subnets in its routing table.

5. R2 sends a summarized update to R1 for the 172.16.0.0/16 network.

6. R1 installs the summarized route for 172.16.0.0/16 via R2.

7. R1 receives a packet for 172.16.4.10. Because R1 has a route for 172.16.0.0/16 via R2, it forwards the packet to R2.

8. R2 receives the packet with the destination address 172.16.4.10 from R1. The packet does not match any specific route so using the default route in its routing table R2 forwards the packet back to R1.

9. The packet for 172.16.4.10 loops between R1 and R2 until the TTL expires and the packet is dropped.

Figure shows a network where router one is connected to i. s. p. and router two. Router two summarized the one seventy two dot sixteen dot zero dot zero and sent it to router one. Router two also has a default static route pointing to router one. As a result of the summary route, router one sends packets destined for one seventy two dot sixteen dot four dot ten to router two. As a result of its default static route, router two forwards these packets back to router one. This is a routing loop.
