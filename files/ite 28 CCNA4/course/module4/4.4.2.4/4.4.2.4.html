Troubleshooting IPv4 ACLs - Example 4

In the figure, host 192.168.30.12 is able to Telnet to connect to 192.168.31.12, but company policy states that this connection should not be allowed. Output from the show access-lists 140 command indicate that the permit statement has been matched.

Solution - Host 192.168.30.12 can use Telnet to connect to 192.168.31.12 because there are no rules that deny host 192.168.30.12 or its network as the source. Statement 10 of access list 140 denies the router interface on which traffic enters the router. The host IPv4 address in statement 10 should be 192.168.30.12.

The figure is another example of an error in an A.C.L. and uses the same reference topology. This figure shows the output from the following command run on R-3, show access hyphen lists 140. Access list number 140 is shown. This particular A.C.L. has 2 statements. The first is 10 deny t.c.p. 192 dot 168 dot 30 dot 1 any e.q. telnet. The second line is 20 permit i.p. any any and has had 5 matches. The A.C.L. should be denying the host address, which is 192 dot 168 dot 30 dot 12 slash 24, but instead it is blocking 192 dot 168 dot 30 dot 1. So the error here is the wrong source address.
